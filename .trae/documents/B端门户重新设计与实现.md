# B端门户重新设计与实现

## 需求分析
用户期望的B端门户是企业用户用于管理其所属建筑的平台，需要包含以下核心功能：
1. 建筑管理：查看所管理的建筑列表
2. 历史防水情况：查看建筑的防水维修历史
3. 报修功能：及时提交报修申请
4. 施工情况：查看当前施工进度
5. 工程准备情况：了解工程前期准备
6. 工程评价：对已完成工程进行评价
7. 合同管理：查看合同信息
8. 发票管理：查看发票详情
9. 付款情况：查看付款记录

## 现有问题
当前B端门户只有批量报修、合同项目管理、台账与对账三个功能，与用户需求差距较大，需要重新设计。

## 实现计划

### 1. 修改B端布局与导航
- 更新 `src/views/b-portal/Layout.vue`，重新设计底部导航栏，包含以下功能入口：
  - 首页/建筑列表
  - 报修中心
  - 施工管理
  - 财务管理
  - 个人中心

### 2. 创建新页面

#### 建筑管理
- `BuildingManagement.vue`：建筑列表页面，展示企业用户所管理的建筑
- `BuildingDetail.vue`：建筑详情页面，包含建筑基本信息和历史防水情况

#### 报修中心
- `RepairCenter.vue`：报修管理页面，包含报修记录列表和新增报修功能
- `RepairDetail.vue`：报修详情页面，查看报修处理进度

#### 施工管理
- `ConstructionManagement.vue`：施工项目列表，查看所有施工项目
- `ConstructionDetail.vue`：施工详情页面，包含施工进度、工程准备情况等
- `ProjectEvaluation.vue`：工程评价页面，对已完成工程进行评价

#### 财务管理
- `ContractManagement.vue`：合同列表页面，查看所有合同
- `ContractDetail.vue`：合同详情页面
- `InvoiceManagement.vue`：发票列表页面
- `PaymentManagement.vue`：付款记录页面

#### 个人中心
- `UserCenter.vue`：企业用户个人信息管理

### 3. 更新路由配置
- 修改 `src/router/index.js`，添加新创建的B端页面路由

### 4. 移动端适配
- 确保所有新页面都采用移动端优先设计，适配不同屏幕尺寸
- 保持与现有C端和师傅端一致的设计风格

## 技术实现
- 采用Vue 3 Composition API
- 使用Element Plus组件库
- 保持与现有系统一致的代码风格和设计规范
- 确保页面响应式设计，适配移动端

## 预期效果
- B端门户将完全符合用户需求，提供企业用户管理建筑、报修、查看施工情况等完整功能
- 页面设计简洁易用，移动端适配良好
- 与现有系统无缝集成，保持一致的用户体验

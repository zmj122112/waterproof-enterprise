## C端客户端页面迁移到waterproof-mini计划

### 1. 项目结构分析
- **waterproof-mini**：基于uni-app的小程序项目，现有页面包括首页、报修页和个人中心页
- **C端客户端页面**：位于`/waterproof-enterprise/frontend-prototypes/src/views/c-mini`，包含24个Vue页面

### 2. 页面对应关系
| waterproof-mini现有页面 | c-mini对应页面 |
|------------------------|----------------|
| pages/index/index      | Home.vue       |
| pages/repair/estimate  | RepairEstimate.vue |
| pages/user/index       | UserCenter.vue |

### 3. 迁移顺序和优先级

#### 第一阶段：核心页面迁移
1. **首页**：将Home.vue迁移到pages/index/index.vue，替换现有页面
2. **报修页**：将RepairEstimate.vue迁移到pages/repair/estimate.vue，替换现有页面
3. **个人中心**：将UserCenter.vue迁移到pages/user/index.vue，替换现有页面

#### 第二阶段：关联页面迁移
1. **订单相关**：OrderList.vue → pages/order/list.vue；OrderDetail.vue → pages/order/detail.vue
2. **服务相关**：MasterList.vue → pages/master/list.vue；MasterDetail.vue → pages/master/detail.vue
3. **评价相关**：CustomerReviewList.vue → pages/review/list.vue；CustomerReviewDetail.vue → pages/review/detail.vue

#### 第三阶段：辅助页面迁移
1. **地址管理**：AddressManagement.vue → pages/user/address.vue
2. **设置**：Settings.vue → pages/user/settings.vue
3. **联系客服**：ExclusiveCustomerService.vue → pages/user/service.vue
4. **反馈**：Feedback.vue → pages/user/feedback.vue
5. **其他页面**：根据功能和导航关系逐步迁移

### 4. 资源处理
- **图片资源**：
  1. 查找c-mini页面中引用的所有图片资源
  2. 将图片资源复制到waterproof-mini的static目录
  3. 调整页面中图片引用路径，确保指向正确的static目录
  4. 处理相对路径和绝对路径，确保在小程序中可以正确访问

### 5. 页面修改规则
- **标签替换**：将所有`<div>`替换为`<view>`，`<span>`替换为`<text>`，`<img>`替换为`<image>`
- **组件替换**：移除所有Element Plus组件，替换为uni-app原生组件
- **样式调整**：将px单位替换为rpx，调整样式和布局适配小程序
- **路由替换**：移除Vue Router依赖，使用uni-app导航方法
- **事件调整**：移除小程序不支持的事件（如mouseenter、mouseleave）
- **图片路径**：调整图片路径，确保在小程序中正确访问

### 6. 实现步骤
1. **资源准备**：复制c-mini页面中使用的图片资源到waterproof-mini的static目录
2. **目录结构创建**：根据迁移计划创建必要的目录
3. **页面复制**：将c-mini页面复制到对应目录
4. **页面修改**：按照修改规则逐个修改页面
5. **路由配置**：更新pages.json，添加新页面路由
6. **功能测试**：测试页面功能和导航
7. **样式优化**：调整样式适配不同设备

### 7. 注意事项
- 保持项目结构清晰，便于维护
- 确保所有页面符合uni-app和微信小程序规范
- 测试页面在不同设备上的显示效果
- 确保导航逻辑正确，页面之间可以正常跳转
- 注意图片资源的路径和大小，优化小程序性能
- 逐步迁移，避免一次性修改过多文件导致问题

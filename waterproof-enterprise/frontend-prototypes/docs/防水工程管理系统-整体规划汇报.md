# 防水工程管理系统 - 整体规划汇报

---

## 一、项目概述

### 项目背景与目标
- **背景**：传统防水工程管理方式效率低下，信息不透明，难以满足现代防水工程管理需求
- **目标**：打造一个覆盖全流程的防水工程管理系统，实现从报修到完工的数字化管理

### 系统架构与技术栈
- **前端架构**：Vue 3 + Element Plus + Vue Router + Pinia
- **后端架构**：待设计
- **数据库**：待设计
- **开发工具**：Vite + VS Code

### 核心功能模块
- **C端小程序**：个人用户报修、预约、工单跟踪
- **B端门户**：企业用户建筑管理、报修、施工监控
- **管理系统**：内部工单分配、资源管理、数据分析
- **师傅端**：现场施工记录、完工确认

---

## 二、用户角色与业务场景画像

### 1. C端个人用户
**角色描述**：居住在住宅或使用商业空间的个人用户，遇到防水问题需要及时处理

**业务场景**：
- 在家中发现卫生间渗水，通过小程序快速报修
- 预约专业师傅上门勘察
- 查看工单处理进度
- 对完成的防水工程进行评价

**核心需求**：
- 报修流程简单快捷
- 能实时了解处理进度
- 服务质量有保障
- 价格透明合理

### 2. B端企业用户
**角色描述**：管理多个建筑或物业的企业用户，需要系统化管理防水维护工作

**业务场景**：
- 查看所管理建筑的历史防水情况
- 批量提交报修申请
- 跟踪施工项目进度
- 查看合同和付款记录
- 对工程进行评价和验收

**核心需求**：
- 建筑防水情况可视化
- 高效的报修管理
- 透明的施工过程
- 完善的财务管理
- 数据统计与分析

### 3. 管理系统用户
**角色描述**：防水工程公司内部管理人员，负责工单分配、人员管理、物料管理等

**业务场景**：
- 接收并分配工单
- 管理客户信息
- 安排施工人员和物料
- 监控项目进度
- 处理财务结算
- 生成统计报表

**核心需求**：
- 高效的工单管理
- 完善的资源调度
- 实时的项目监控
- 准确的财务核算
- 数据驱动决策

### 4. 师傅端用户
**角色描述**：现场作业的防水施工师傅，负责工单接收和施工记录

**业务场景**：
- 接收新工单并前往现场
- 进行防水勘察并记录
- 执行施工作业
- 上传施工照片和记录
- 确认完工并等待验收

**核心需求**：
- 清晰的工单信息
- 便捷的现场记录
- 高效的施工管理
- 及时的完工确认

## 三、核心业务流程

### 1. C端报修流程
1. 用户发现防水问题
2. 打开小程序提交报修
3. 系统分配师傅
4. 师傅上门勘察
5. 双方确认方案和价格
6. 师傅进行施工
7. 施工完成验收
8. 用户评价

### 2. B端建筑管理与报修流程
1. 企业用户登录B端门户
2. 查看建筑列表和历史防水情况
3. 提交报修申请
4. 管理系统接收并分配工单
5. 施工师傅现场施工
6. 企业用户查看施工进度
7. 施工完成验收
8. 企业用户评价

### 3. 管理系统业务流程
1. 接收C端或B端报修
2. 分配工单给合适的师傅
3. 跟踪施工进度
4. 管理物料和人员
5. 处理财务结算
6. 生成统计报表

### 4. 师傅端施工流程
1. 接收新工单
2. 现场勘察并记录
3. 制定施工方案
4. 执行施工作业
5. 上传施工记录和照片
6. 确认完工
7. 等待验收

## 四、系统功能模块

### 1. C端小程序
- **首页**：快速报修入口、服务介绍
- **报修/估价**：提交报修申请、获取报价
- **预约上门**：选择服务时间
- **工单进度**：实时查看工单处理状态
- **个人中心**：用户信息、历史记录

### 2. B端门户
- **建筑管理**：建筑列表、建筑详情、历史防水记录
- **报修中心**：报修记录管理、新增报修
- **施工管理**：施工项目列表、施工详情、工程评价
- **财务管理**：合同管理、发票管理、付款记录
- **个人中心**：企业信息、修改密码

### 3. 管理系统
- **工单管理**：工单列表、工单分配、工单详情
- **客户管理**：客户信息、客户分类
- **人员管理**：员工信息、角色权限
- **物料管理**：物料库存、物料采购
- **合同管理**：合同列表、合同详情
- **财务管理**：收支管理、财务报表
- **数据统计**：业务数据、趋势分析

### 4. 师傅端
- **工单接收**：新工单列表、工单详情
- **施工记录**：现场勘察、施工过程记录
- **完工确认**：完工验收、照片上传

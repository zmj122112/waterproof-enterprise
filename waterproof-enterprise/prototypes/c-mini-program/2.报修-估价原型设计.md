# C端小程序 - 报修/估价原型设计

## 一、页面概述

### 1.1 页面定位
- 提供用户快速报修和智能估价功能
- 引导用户填写漏水情况和相关信息
- 生成初步报价范围，帮助用户决策
- 连接用户与服务团队，形成服务闭环

### 1.3 交互设计

#### 1.3.1 顶部导航栏
- **返回**：点击返回上一页
- **取消**：点击取消当前操作，返回首页

#### 1.3.2 漏水类型选择
- **漏水类型卡片**：点击选择对应的漏水类型，选中状态高亮显示
- **动画效果**：点击时缩放+颜色变化效果

#### 1.3.3 漏水情况描述
- **文字描述输入框**：支持多行输入，实时显示输入字数
- **图片上传区域**：
  - 点击上传按钮，调用手机相册或相机
  - 支持图片预览和删除
  - 上传成功后显示缩略图
  - 动画效果：上传成功后显示旋转对勾动画

#### 1.3.4 智能估价结果
- **估价结果**：根据用户输入自动计算，实时更新
- **服务保障说明**：滚动显示，支持点击查看详情

#### 1.3.5 操作按钮
- **预约上门检测**：点击进入预约上门流程
- **重新估价**：点击清空当前输入，重新开始估价
- **动画效果**：
  - 预约按钮：点击时变色+缩放效果
  - 重新估价按钮：点击时显示刷新动画

### 1.4 设计规范

#### 1.4.1 颜色方案
- 主色调：蓝色 (#1890ff) - 代表专业、信任
- 辅助色：橙色 (#ff7a45) - 代表活力、行动
- 成功色：绿色 (#52c41a) - 代表估价成功
- 背景色：浅灰色 (#f5f5f5)
- 文字色：深灰色 (#333333)、中灰色 (#666666)、浅灰色 (#999999)

#### 1.4.2 字体方案
- 标题：18px - 20px，粗体
- 正文：14px - 16px，常规
- 辅助文字：12px，常规

#### 1.4.3 图标方案
- 使用简洁、直观的线性图标
- 图标颜色与文字颜色保持一致
- 交互元素图标添加反馈效果

### 1.6 技术实现建议

#### 1.6.1 前端框架
- 使用微信小程序原生开发
- 或使用 Taro、uni-app 等跨端框架

#### 1.6.2 交互实现
- 使用微信小程序的 animation API 实现动画效果
- 使用 wx.chooseImage API 实现图片上传
- 使用 wx.navigateTo API 实现页面跳转
- 使用 wx.showToast、wx.showLoading 等 API 实现反馈效果

#### 1.6.3 数据来源
- 智能估价算法：调用后台估价服务API
- 服务保障信息：后台配置

## 二、原型使用说明

### 2.1 如何查看原型
- 使用微信开发者工具打开原型文件夹

- 填写文字描述，上传图片，查看估价结果
- 点击预约上门检测，查看跳转效果
- 点击重新估价，查看清空效果
